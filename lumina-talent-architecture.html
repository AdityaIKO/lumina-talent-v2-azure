🔴 BOTTLENECK #01
Azure AI — Single Point of Failure
OCR untuk KYC, Co-Pilot matchmaking, dan pembuatan laporan keuangan semuanya bergantung pada Azure AI. Jika service ini down, tiga fitur inti platform berhenti sekaligus. Risiko availability sangat tinggi untuk platform komersial.
🔴 BOTTLENECK #02
Cosmos DB Tanpa Pemisahan Tier
Semua data — pesan chat, profil freelancer, transaksi keuangan — bermuara ke satu database yang sama. Data keuangan seharusnya di-isolasi dengan access control lebih ketat; log percakapan bisa pakai cold storage yang lebih murah.
🟠 RISIKO LOGIKA #03
Trigger "Finished" Escrow Tidak Jelas
Siapa yang menekan tombol "selesai"? Employer? Freelancer? Keduanya harus setuju? Jika tidak terdefinisi, ini bisa jadi celah fraud atau sumber konflik yang tidak terselesaikan.
🟠 MISSING FLOW #04
Tidak Ada Alur Penolakan Proposal
Flowchart hanya mendefinisikan "Accept Proposal" — tidak ada flow untuk reject, counter-offer, atau feedback ke freelancer yang proposalnya tidak lolos. Freelancer tidak tahu mengapa ditolak.
🟠 MISSING FLOW #05
KYC Tidak Punya Jalur Re-verification
Jika dokumen KTP/NIB ditolak AI atau masa berlaku KYC habis, tidak ada alur untuk submit ulang. Pengguna akan tersangkut tanpa jalan keluar yang jelas.
🟠 MISSING FLOW #06
Sistem Notifikasi Tidak Terdefinisi
Tidak ada notification layer eksplisit. Event krusial seperti "proposal diterima", "pembayaran masuk", atau "status kontrak berubah" harus dikomunikasikan secara real-time ke pengguna.
🟠 MISSING FLOW #07
Post-Project Flow Tidak Ada
Rating dan review disebutkan sebagai data di dashboard, tetapi tidak ada alur yang mendefinisikan kapan dan bagaimana rating diberikan setelah proyek selesai.
🟢 IMPROVEMENT #08–11
4 Saran Improvement Utama
AI fallback mechanism jika Azure down → manual queue. Dispute resolution system dengan mediator + SLA. Database separation: financial / chat / profile. Notification layer event-driven: push notif, email, in-app alert.

Jika Azure AI OCR tidak tersedia: dokumen masuk antrian manual review admin. Status ditampilkan ke user: "Under Review — estimasi 1×24 jam". Notifikasi email ketika review selesai.

AIFIXNEW
▾

⚠

✦
NEW — Counter-offer (Opsional)
Tambah opsi "Counter Offer" — employer bisa mengajukan negosiasi budget atau timeline sebelum accept/reject penuh. Freelancer menerima notifikasi dan bisa Accept / Decline counter-offer tersebut.
2.7 Ekosistem Escrow & Proyek Berjalan
DBAIFIXNEW
▾
Deposit Escrow Fund: klien menyetorkan dana yang ditahan.
Status: Hiring → In Progress → Completed / Disputed.
Payment Gateway: Midtrans/Xendit/QRIS (Lokal), Stripe & Wise API (Internasional).
Transaksi & Report → Azure AI + Document Intelligence untuk generate laporan keuangan.
⚠
FIX — Trigger "Finished" yang Jelas
Alur konfirmasi selesai: (1) Freelancer klik "Mark as Done". (2) Employer menerima notifikasi + tombol "Confirm Completion" atau "Raise Dispute". (3) Jika Employer confirm → status "Completed" → dana escrow dicairkan ke freelancer. (4) Jika tidak ada respons dalam 3×24 jam → auto-complete. (5) Jika Raise Dispute → masuk alur Dispute Resolution.
✦
NEW — Dispute Resolution System
Jika terjadi sengketa: (1) Kedua pihak mengajukan klaim + bukti. (2) Admin platform bertindak sebagai mediator. (3) SLA: keputusan dalam 5 hari kerja. (4) Keputusan admin bersifat final — dana dicairkan ke pihak yang berhak atau dibagi sesuai keputusan. (5) Rekam jejak dispute tersimpan untuk audit.
Saran Improvement Teknis
4 area perbaikan arsitektur yang disarankan untuk meningkatkan reliability, security, dan user experience platform.
🔄
AI Fallback Mechanism
Pastikan platform tetap berfungsi ketika Azure AI tidak tersedia, dengan fallback ke manual review oleh admin.
01
Deteksi Azure AI timeout/error (threshold: 10 detik).
02
Dokumen KYC masuk antrian manual review (admin queue).
03
Notifikasi user: "Sedang diproses manual — estimasi 1×24 jam".
04
Admin dashboard untuk review dokumen dan approve/reject.
05
Notifikasi email ketika keputusan sudah dibuat.
⚖️
Dispute Resolution System
Mekanisme terstruktur untuk menangani konflik antara employer dan freelancer, dengan mediator dan timeline SLA yang jelas.
01
Employer klik "Raise Dispute" → form pengajuan klaim + upload bukti.
02
Freelancer menerima notifikasi dan bisa submit counter-klaim.
03
Admin mediator ditugaskan otomatis (SLA: respons 1×24 jam).
04
Keputusan final dalam 5 hari kerja — dana dicairkan atau dibagi.
05
Rekam jejak dispute diarsip untuk audit dan referensi.
🗄️
Database Separation (3-Tier)
Pisahkan data berdasarkan sensitivitas dan pola akses untuk keamanan, performa, dan efisiensi biaya.
T1
Primary DB — Profil, matching, job posting. Akses tinggi, latensi rendah.
T2
Financial DB — Transaksi, escrow, tagihan. Isolated, encrypted at rest & in transit, audit log wajib.
T3
Cold Storage — Chat history, log percakapan. Akses rendah, biaya penyimpanan murah (Azure Blob / Archive).

🔔
Event-Driven Notification Layer
Tambahkan layer notifikasi yang terintegrasi dengan perubahan status di seluruh platform untuk komunikasi real-time ke pengguna.
01
Setiap perubahan status di DB memicu event ke Azure Service Bus.
02
Notification service membaca event dan menentukan channel pengiriman.
03
Push notification — mobile via Firebase Cloud Messaging.
04
Email — via SendGrid / Azure Communication Services.
05
In-app alert — bell icon, badge count, notification center.
